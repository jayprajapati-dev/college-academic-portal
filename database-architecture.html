info Database Overview

This system uses MongoDB as the primary database. MongoDB is a NoSQL database that stores data in structured collections based on academic hierarchy and workflows. Data is organized by role, semester, subject, and academic activities.

security Role-Based Database Access

admin_panel_settings Administrator

  • ✓ Create/Update/Delete Branches
  • ✓ Create/Update Semesters
  • ✓ Add HOD & Teacher Accounts
  • ✓ Full database access (Read/Write/Update)
  • ✓ View all system data

person_4 Head of Department (HOD)

  • ✓ Assign Teachers to Subjects
  • ✓ Add Subject-Level Details (marks, exam types)
  • ✓ View Attendance & Task data
  • ✓ Monitor branch performance
  • ✗ Cannot create branches or users

person_3 Teacher

  • ✓ Add Tasks & Assignments
  • ✓ Mark Attendance
  • ✓ Update Task Submission Status
  • ✓ Enter Subject Marks & Exam Details
  • ✓ View student data for assigned subjects
  • ✗ Cannot create subjects or branches

person Student

  • ✓ View assigned Subjects & Tasks
  • ✓ View Attendance records
  • ✓ View Marks & Exam details
  • ✓ Mark Task as Done/Not Done
  • ✗ Cannot modify database directly
  • ✗ Cannot access other students' data

trending_up Real Data Flow Examples

event_available Example 1: Teacher Marks Attendance

Step 1: Teacher opens Attendance page in UI

Step 2: Click "Mark Attendance" button

Step 3: API receives request → Express server processes

Step 4: MongoDB attendance collection updated with date, status

Step 5: Percentage auto-calculated

Step 6: UI shows "Marked Successfully"

Step 7: Student dashboard updates automatically (Real-time or on refresh)

assignment Example 2: Teacher Assigns Task

Step 1: Teacher goes to Tasks → Create New Task

Step 2: Fills: Title, Description, Due Date, Subject

Step 3: Clicks "Save Task" → API request sent

Step 4: MongoDB tasks collection gets new entry

Step 5: Task is now visible in student's SubjectHub

Step 6: Student marks "Done" → taskStatus updated (Pending)

Step 7: Teacher reviews → Marks "Completed" or "Not Completed"

grade Example 3: Teacher Enters Exam Marks

Step 1: Teacher goes to "Subject" → Select exam type (Midterm, Final, etc.)

Step 2: Enters marks for all students

Step 3: Clicks "Save Marks" → API processes

Step 4: MongoDB subjectDetails collection updated

Step 5: Student dashboard shows marks immediately

Step 6: System auto-calculates total percentage
