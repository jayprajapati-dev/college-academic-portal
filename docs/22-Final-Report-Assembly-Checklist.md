# Final Report Assembly Checklist

## Stage 1: Content Freeze
- [ ] All modules selected for report are finalized in current build
- [ ] Out-of-scope modules are clearly marked
- [ ] Real data collection sheet is fully filled

## Stage 2: Chapter Drafting
- [ ] Introduction to conclusion drafted as per chapter guide
- [ ] API chapter uses real endpoint samples
- [ ] Database chapter matches current schema
- [ ] Testing chapter includes real test outcomes

## Stage 3: Figures and Tables
- [ ] All mandatory screenshots captured
- [ ] Figure IDs mapped to chapters
- [ ] Captions added in required style
- [ ] List of Figures generated
- [ ] List of Tables generated

## Stage 4: Formatting and Pagination
- [ ] Same font and heading style across whole report
- [ ] Page numbers visible and continuous
- [ ] No broken heading at page bottom
- [ ] No empty large gaps caused by manual spacing
- [ ] No chapter starts with leftover lines from previous section

## Stage 5: Authenticity Audit
- [ ] No fake statistics or fake claims
- [ ] No AI filler phrases
- [ ] No copied irrelevant content from old report
- [ ] All screenshots are from your project
- [ ] College and student details verified

## Stage 6: Final Export
- [ ] TOC updated after final pagination
- [ ] PDF generated and opened on another device for verification
- [ ] Print preview checked (A4)
- [ ] Final filename format applied
  - Example: Smart_College_Academic_Portal_Final_Report_<EnrollmentNo>.pdf

## Stage 7: Submission Set
- [ ] Final report PDF
- [ ] PPT
- [ ] Source code package
- [ ] Database/schema supporting files
- [ ] Demo checklist
