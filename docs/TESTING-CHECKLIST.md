# Testing Checklist

Use this checklist for manual QA before demos and submissions.

## Authentication
- Login with email and mobile
- Forgot password flow with security question
- First-login password change

## User Management
- Admin: create HOD, Teacher, Student
- HOD: create Teacher and Student in own branch
- Block and unblock users
- Admin access mode toggle (HOD/Teacher)

## Academic Structure
- Create, edit, delete branches
- Create, edit, delete semesters
- Create, edit, delete subjects
- Assign teachers to subjects

## Notices
- Create notice for all roles
- Create notice for selected roles
- Verify HOD and Teacher scoping
- Edit and delete notice

## Tasks
- Teacher creates task
- Student submits response
- Teacher views submissions
- Teacher updates status
- Reminder notifications appear before due date

## Attendance
- Mark attendance (Present/Absent/Late)
- Lecture vs Lab-Tutorial session types
- Edit an attendance session
- Student attendance view

## Timetable, Exams, Library, Materials
- Timetable create and view
- Exam schedule create and view
- Exam results view
- Library book add and view
- Materials upload and download

## Notifications
- Notification bell shows new items
- Action links navigate to correct pages
- Mark notifications as read

## Responsiveness
- Test at 375px, 768px, and 1366px widths
- Tables should be scrollable on mobile
- Modals should fit viewport
