# Report Formatting and Writing Standards

## 1. Professional Formatting Rules
- Use one font family consistently in body text
- Keep heading hierarchy fixed (H1, H2, H3)
- Do not change alignment style randomly between chapters
- Keep page number visible on every page except cover (if institute rule says so)
- Use consistent line spacing and paragraph spacing

## 2. Page Break Discipline
- Do not keep chapter heading at page bottom without content
- Move heading to next page if fewer than 3 lines can follow
- Do not create pages with only 1 to 2 lines unless unavoidable
- Avoid breaking a table row across pages
- Keep figure and caption on the same page

## 3. Language Quality Rules
- Use simple formal academic English
- Avoid AI-style claims such as "revolutionary", "state-of-the-art" unless proven
- Avoid repeated marketing words
- Prefer evidence-based statements
  - Good: "Role-based routing is implemented using JWT and middleware"
  - Avoid: "The system is very secure and very advanced"

## 4. Data Integrity Rules
- Every module claim must have one of the following:
  - Code reference
  - API sample
  - Screenshot
  - Test case result
- If a feature is partial, write "Partially implemented" with reason
- Do not add future features in implemented sections

## 5. Symbol and Notation Rules
- Use one bullet style consistently
- Use one table style consistently
- Use consistent quote style for endpoint strings
- Use monospaced text for API routes and code snippets

## 6. Chapter End Checklist
- Section numbering is correct
- Figures/tables are numbered correctly
- Cross-references point to valid chapter/figure
- No dummy text remains
- No placeholder like "Lorem ipsum" or "TBD" remains

## 7. Final Pre-Submission Checks
- Spell check completed
- Grammar check completed
- Table of contents page numbers refreshed
- List of Figures and List of Tables refreshed
- PDF export tested on another system
