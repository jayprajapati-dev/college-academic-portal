# Report Screenshot and Figure Plan

## Objective
Capture clean, real, and chapter-mapped screenshots so the report looks professional and verifiable.

Detailed module-by-module mapping is available in:
- docs/24-Screenshot-Master-Mapping.md

## Screenshot Naming Standard
- Format: SS-XX-Module-Action.png
- Example: SS-23-Timetable-Weekly-View.png
- Store in one folder before report insertion: docs/ppt-parts or a dedicated report-assets folder

## Capture Rules
- Use same browser zoom for all captures (100%)
- Use clean data (no debug logs visible)
- Avoid cropped UI where context is lost
- Capture both success and validation/error cases where important
- Do not include personal secrets or token values

## Mandatory Screenshot Set
1. Landing page overview
2. Login page
3. Role-specific dashboards (Admin, HOD, Teacher, Student)
4. Subject module screens
5. Notice create and list screens
6. Task create, submit, and review screens
7. Timetable weekly controls and management table
8. Library/materials screens
9. Profile/settings or first-login setup
10. API response sample evidence
11. Database evidence (collection/records snapshot without sensitive data)

## Figure Caption Style
- Format: Figure X.Y: <Short Action-focused Caption>
- Example: Figure 6.3: Teacher dashboard with task creation panel

## Figure Placement Rules
- Place screenshot immediately after first mention in text
- Keep minimum 2 lines of explanation before each figure
- Do not place one figure across two pages
- Keep caption style and alignment consistent

## Screenshot Quality Checklist
- Text is readable at print scale
- Relevant UI controls are visible
- Data shown is realistic and project-related
- Caption and chapter reference added
- Figure number updated in List of Figures
